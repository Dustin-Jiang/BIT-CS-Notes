## 人力资源计划

有效的发挥每个参与项目人员作用的过程。

### 项目组织形式 ###重点
| 组织类型                   | 人员安排特点                                        | 优点                                | 缺点                                      |
| ---------------------- | --------------------------------------------- | --------------------------------- | --------------------------------------- |
| **项目型（Project-based）** | 员工按项目组集中分配，项目经理拥有较大权力，团队成员由不同职能部门抽调。          | - 项目目标明确，责任清晰 - 决策迅速，执行效率高 - 团队凝聚力强 | - 资源可能浪费或重复 - 成员对职能部门联系弱 - 项目结束后团队解散，经验流失 |
| **矩阵型（Matrix）**        | 员工同时隶属于职能部门和项目组，存在“双重汇报”机制。                   | - 平衡职能与项目需求 - 促进跨部门沟通 - 资源利用较灵活     | - 权责不清，可能出现冲突 - 管理复杂，决策慢 - 成员压力大，汇报负担重    |
| **职能型（Functional）**    | 员工按专业技能分部门，如研发、市场、财务等，由部门主管统一管理。              | - 专业发展明确 - 管理简便 - 资源集中，重复劳动少        | - 缺乏项目灵活性 - 部门间沟通受限 - 创新和跨职能协作不足          |
| **敏捷型（Agile / Scrum）** | 员工组成小型跨职能团队（Scrum Team），按迭代周期执行任务，强调自组织和快速反馈。 | - 快速响应变化 - 高度协作与透明 - 持续改进和客户导向      | - 对管理和团队自律要求高 - 大规模组织实施困难 - 角色职责可能模糊      |

### 人员配备管理计划

- 人员获取、时间表、释放标准
- 培训需求、认可和奖励、安全性

### 责任分配矩阵（RAM）###重点

- **RACI 模型**：
  - R（Responsible）：负责执行
  - A（Accountable）：批准决策
  - C（Consulted）：提供意见
  - I（Informed）：被告知结果

### 团队建设 ###重点

- **形成期→振荡期→规范期→发挥期→结束期**
- 项目经理不仅是管理者，更是领导者
- 激励理论：马斯洛需求层次、海兹伯格双因素理论、X-Y 理论

### 敏捷团队

- 黄金人数 5-9 人
- 自组织团队
- 仆人式领导
- 角色：Product Owner、Scrum Master、开发团队

## 沟通计划

确保项目信息及时、适当产生、收集、传播、保存。

### 沟通模型

- **编码→信息→媒介→（噪声）→解码**
- 确认收讫：表示收到，不一定同意
- 回应：理解并回复

### 沟通方式

- 书面沟通和口头沟通
- 语言沟通和非语言沟通
- 正式沟通和非正式沟通
- 单向沟通和双向沟通
- 网络沟通

### 项目干系人管理 ###重点

- **识别干系人**：找出受项目影响或影响项目的个人/组织
- **干系人分析**：权力/利益方格、权力/影响方格、凸显模型
- **干系人管理原则**：内外有别、充分理解需求、解决冲突优先考虑客户利益

### 沟通计划

- 确定谁需要什么信息、何时需要、如何获取
- 一般在项目初期完成，后续定期审查调整

### 沟通方法

- **交互式沟通**：会议、电话等
- **推式沟通**：邮件、报告等
- **拉式沟通**：门户网站、知识库等

### OKR（目标和关键结果）###重点

- 一套组织、团队与个人皆适用的目标管理方法
- 运用企业：Intel、Google、Oracle、百度、腾讯、华为
- 目标来源：使命愿景分解、团队讨论、创新要求等
